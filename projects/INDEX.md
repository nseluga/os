# projects/

This folder holds one index README per project — not the code itself, but a snapshot of status, goals, and current direction. Each README follows the `_TEMPLATE.md` frontmatter contract; the frontmatter is the data interface that os tooling reads.

Projects span multiple domains: MLB analytics (player modeling, pitch evaluation), web services (studios, client platforms), and personal infrastructure (os dashboard, evaluation frameworks).

## Projects

- `batting-average-ability` — Separates MLB hitter batting-average skill from luck via ML and statistical modeling
- `bcns` — Software studio (Nate + Brandon Chung) — platform repo: marketing site, shared packages, client template
- `bcns-client-coventry` — Coventry Painting — first hosted-web client: ops platform (leads → quotes → dispatch → closeout) + sub-facing PWA
- `bcns-client-delucas` — DeLuca's restaurant management — local-first Electron desktop app; first client (build-and-hand-off era)
- `bcns-client-l2detailz` — Converting L2 Detailz's static site into a web app with admin dashboard
- `hitter-embedding` — Conditional-query hitter embedding on Statcast signals; research target SSAC27, abstract due Oct 1
- `nba-shot-value` — Predicting expected point value of NBA shots from play-by-play context
- `os` — Personal operating system + its dashboard app — the single local UI for os and project management
- `os-evals` — Ablation ladder measuring which layers of ~/os earn their keep
- `patio` — Full-stack social betting app for backyard games
- `pitcher-injury-risk` — MLB pitcher health modeling — injury probability, severity, time-to-injury, Injury Risk+
- `portfolio-website` — Personal portfolio site — software engineering, sports analytics, AI tooling
