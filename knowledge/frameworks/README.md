# frameworks/

Mental models, methods, and reusable ways of thinking.

Durable, project-independent knowledge: decision frameworks, checklists,
principles, and processes worth applying more than once.

## Files

- `research-standards.md` — challenge checklist, pushback taxonomy, reference policy, and decision/notebook log formats for research work
- `plan-md.md` — how to write PLAN.md files for dev-team and layout-loop runs: schema, field explanations, `done when:` criteria guidance, stop marker
- `progress-md.md` — how PROGRESS.md is structured, who writes it, row conventions for dev-team-auto and layout-loop, and how it relates to PLAN.md
- `notebook-code-standards.md` — shared coding standards for all Jupyter notebook research work: output formatting, comments, docstrings, cell scope, named constants, import hygiene, dead cell cleanup, variable naming
