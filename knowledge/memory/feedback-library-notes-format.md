One notes file per document. Do NOT create a single aggregated NOTES.md for a
multi-paper folder.

**Why:** User interrupted and corrected this explicitly when I tried to combine
all four baseball papers into one pitch-strategy/NOTES.md.

**How to apply:** When triaging library/_inbox items, create
`<slug>.md` alongside each `<slug>.pdf` — same folder, same name, separate
files.

Also: never write placeholder/scaffolded content before actually reading the
source. Read first, then write the summary. User also stopped an earlier
attempt where I invented section headers without opening the PDFs.
