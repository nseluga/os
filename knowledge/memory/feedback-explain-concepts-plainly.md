Lead with a plain-English explanation any time a Claude Code concept is new in this session — don't just name it and move on.

**Why:** User asked "what do these mean?" immediately after receiving a cheat sheet of efficiency rules (batch independent calls, use Read not Bash(cat), etc.). The rules were correct but assumed familiarity with terms the user hadn't encountered yet.

**How to apply:** When introducing "batching," "dedicated file tools," "subagents," "hooks," or similar — one sentence of plain English before the technical detail. Don't do this for concepts already demonstrated as familiar in the current session.
