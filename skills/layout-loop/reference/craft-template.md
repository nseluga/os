# Craft layer template — Nate's invariant design taste

This file is the backbone of the `layout-loop` skill. It holds what is **true of Nate's
taste regardless of brand or client** — the "uniquely Nate" signal that rides along on
every project, even a client dashboard wearing someone else's colors.

Fill it by **reacting to real pixels** in a dedicated cowork extraction session (pull up
5–10 sites Nate loves and a few he hates, and articulate *why*). Do not fill it from cold
description. Every entry should be defensible and, where possible, point to a concrete
example. Principles capture judgment; tokens make it enforceable.

Final home: `~/os/knowledge/library/design-language/craft.md`

---

## Principles (the judgment — prose)

For each, state the conviction, the *why*, and a reference example (URL or screenshot path).

- **Hierarchy** — how Nate wants the eye led. (e.g. "one clear focal point per view; …")
- **Whitespace & density** — generous vs. compact; where breathing room is non-negotiable.
- **Restraint** — accent usage, ornamentation, when to say no. ("one accent earning its place")
- **Intuitiveness** — what makes a layout feel obvious to a first-time user.
- **Simplicity** — how visual noise gets cut; what counts as a justified element.
- **Motion / interaction feel** — if any (subtle vs. expressive; or none).
- **What Nate reacts against** — the anti-patterns (name specific sites that get it wrong).

## Tokens (the application — concrete values)

Only the ones that are invariant across brand. Brand-specific values (palette, brand type)
live in the brand file, not here.

- **Type scale** — ratio / sizes / weights Nate defaults to.
- **Spacing scale** — the rhythm (e.g. 4/8px base, the steps).
- **Radius** — sharp vs. rounded, and how much.
- **Borders / shadows / elevation** — hairline vs. shadow vs. flat.
- **Grid / breakpoints** — column logic; mobile/desktop assumptions.
- **Default density** — comfortable vs. compact baseline.
